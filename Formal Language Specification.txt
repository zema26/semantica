Formal Language Specification
Semantica
Language Spec

A complete formal definition of the Semantica programming language: lexical rules, concrete and abstract syntax, type system, and operational semantics.


Paradigm
Imperative + OOP + Generic

Typing
Static, manifest, nominal

Extension
.sem


1. Lexical Specification

Token classes, reserved words, and comment syntax recognized by the Semantica scanner.

1.1 Reserved Keywords

Construct				Rule / Type					Notes

module | |module				Module open/close delimiter		block keyword

intermodular | |intermodular		Intermodular open/close	block keyword

sub | |sub				Subroutine open/close			block keyword

when | |when			Conditional open/close (≡ if/end if)			block keyword

other | |other			Else branch delimiter (≡ else/end else)		block keyword

repeat | |repeat			While-loop open/close (≡ while/end while)	block keyword

iterate | |iterate			For-loop open/close (≡ for/end for)			block keyword

class | |class				Class open/close				block keyword

template | |template		Generic template open/close	block keyword

link | |link				Module import open/close	block keyword

back | |back				Return value (≡ return)		block keyword

in						Standard input source			I/O keyword

out						Standard output sink			I/O keyword

break					Loop exit					control keyword

true false				Boolean literals				literal

typename				Generic type parameter introducer		generic keyword


1.2 Primitive Types

Construct				Rule / Type				Notes

int						Integer (platform word size)
	
float					IEEE-754 double	

char						Single character	

string					Vector of char			alias for char|n|

bool					Boolean: true | false	

file						File handle				special type


1.3 Operators & Punctuation

Construct	Rule / Type						Notes

>	Assignment (outside condition) / greater-than (inside condition)	context-sensitive

<			Less-than						comparison only

==			Equality	

!=			Inequality	

<= >=		Ordered comparison	

+ - * / %		Arithmetic (standard precedence)
	
++ --		Unary pre/post increment/decrement
	
*			Dereference (generics context)		pointer-style

|			Block delimiter / vector bracket	structural

.			Member access
	
,			Parameter separator	

( )			Expression grouping / initializer

	
1.4 Literals & Comments

EBNF

INTEGER_LIT  ::= [0-9]+
FLOAT_LIT   ::= [0-9]+ '.' [0-9]+
CHAR_LIT    ::= '"' <any char> '"'
STRING_LIT  ::= '"' <any char>* '"'
BOOL_LIT    ::= 'true' | 'false'
IDENT       ::= [a-zA-Z_][a-zA-Z0-9_]*
FILENAME    ::= <any char>+ '.sem'
COMMENT     ::= '//' <any char>* <newline>   (* discarded *)


2. Concrete Syntax

EBNF grammar. ( ... )? optional, ( ... )* zero-or-more, ( ... )+ one-or-more, A | B alternation.

Top Level

EBNF

program         ::= intermodular? module

intermodular    ::= 'intermodular|'
                      link*
                      sub_sig*
                    '|intermodular'

link            ::= 'link|' FILENAME '|link'

sub_sig         ::= 'sub|' params '>' IDENT ':' type '|sub'

module          ::= 'module|' IDENT decl* '|module'


Declarations

EBNF

decl            ::= sub_decl
                  | class_decl
                  | template_decl
                  | var_decl

sub_decl        ::= 'sub|' params '>' IDENT ':' type
                      stmt*
                    '|sub'

class_decl      ::= 'class|' IDENT
                      var_decl*
                      sub_decl*
                    '|class'

template_decl   ::= 'template|' template_param (',' template_param)* '|template'

template_param  ::= 'typename' IDENT

params          ::= ( param (',' param)* )?
                  | '(' param (',' param)* ')'   	(* parenthesised form *)

param           ::= type IDENT ( '|' '|' )?      (* '| |' marks vector param *)


Variable Declaration

EBNF

var_decl        ::= type var_init (',' var_init)*

var_init        ::= IDENT ( '|' expr '|' )?        (* scalar or vector size *)
                    ( '(' initializer ')' )?           		(* optional initializer *)

initializer     ::= literal
                  | '|' expr (',' expr)* '|'           	(* vector literal: (|1,2,3,4|) *)


Statements

EBNF

stmt            ::= var_decl
                  | assignment
                  | when_stmt
                  | repeat_stmt
                  | iterate_stmt
                  | back_stmt
                  | pipeline
                  | expr_stmt
                  | 'break'

assignment      ::= expr '>' lvalue

when_stmt       ::= 'when|' cond stmt* ( 'other|' stmt* '|other' )? '|when'

repeat_stmt     ::= 'repeat|' cond stmt* '|repeat'

iterate_stmt    ::= 'iterate|' var_init iter_step iter_bound stmt* '|iterate'
iter_step       ::= '++' | '--'
iter_bound      ::= ( '<' | '<=' | '>' | '>=' ) expr

back_stmt       ::= 'back|' expr '|back'


Pipeline & I/O

EBNF

pipeline        ::= io_source pipeline_step* io_sink?
                  | expr pipeline_step+ ( '>' 'out' )?

io_source       ::= 'in'
io_sink         ::= 'out'

pipeline_step   ::= '>' lvalue                       (* bind to variable  *)
                  | '>' IDENT                           	(* pass to function  *)
                  | '>' IDENT '.' IDENT                 	(* call member fn    *)

(* Example: in > a, b > Euclid > out  *)

Expressions

EBNF

expr            ::= expr bin_op expr
                  | unary_op expr
                  | primary

primary         ::= literal
                  | lvalue
                  | '(' expr ')'
                  | '*' primary                             (* generic dereference *)

lvalue          ::= IDENT
                  | IDENT '|' expr '|'                  (* vector element a|i| *)
                  | IDENT '.' IDENT                     (* member access N.a   *)
                  | IDENT '.' IDENT                     (* member fn call N.Euclid *)

cond            ::= expr                                    (* boolean-yielding expr *)
                  | expr '!=' expr
                  | expr '==' expr
                  | expr '<'  expr
                  | expr '>'  expr                          (* '>' is comparison here *)
                  | expr '<=' expr
                  | expr '>=' expr

bin_op          ::= '+' | '-' | '*' | '/' | '%' | '==' | '!=' | '<' | '<=' | '>='
unary_op        ::= '-' | '!' | '++' | '--' | '*'

literal         ::= INTEGER_LIT | FLOAT_LIT | CHAR_LIT | STRING_LIT | BOOL_LIT


File System

EBNF

file_decl       ::= 'file' IDENT '(' STRING_LIT ',' file_type ',' file_opts ')'

file_type       ::= 'bin' | 'txt' | 'hex'
file_opts       ::= 'r' | 'w' | 'wr+'

file_stmt       ::= IDENT '.open'
                  | IDENT '.close'
                  | expr '>' IDENT                      (* write: s > f *)
                  | IDENT '>' lvalue                    (* read:  f > s *)
                  | 'not' IDENT '.eof'                  (* eof check in repeat| *)


3. Abstract Syntax

The abstract syntax defines the structure of well-formed Semantica programs as algebraic data types. Concrete surface forms (pipe delimiters, keyword spelling) are erased at this level.

Program & Module

Abstract Syntax

Program     ::= Program {
                    intermodular : Intermodular option,
                    module       : Module
                  }

Intermodular ::= Intermodular {
                    links   : Filename list,          (* imported .sem files *)
                    exports : SubSig  list            (* exported signatures *)
                  }

SubSig      ::= SubSig {
                    params     : Param list,
                    name       : Ident,
                    returnType : Type
                  }

Module      ::= Module {
                    name  : Ident,
                    decls : Decl list
                  }


Declarations

Abstract Syntax

Decl        ::= SubDecl    SubDecl
              | ClassDecl  ClassDecl
              | TmplDecl   TmplDecl
              | VarDecl    VarDecl

SubDecl     ::= SubDecl {
                    params     : Param list,
                    name       : Ident,
                    returnType : Type,
                    body       : Stmt list
                  }

ClassDecl   ::= ClassDecl {
                    name    : Ident,
                    fields  : VarDecl list,
                    methods : SubDecl list
                  }

TmplDecl    ::= TmplDecl {
                    typeParams : Ident list            (* typename identifiers *)
                  }

Param       ::= Param {
                    typ      : Type,
                    name     : Ident,
                    isVector : Bool
                  }

VarDecl     ::= VarDecl {
                    typ     : Type,
                    name    : Ident,
                    size    : Expr option,            (* present → vector *)
                    init    : Init option
                  }

Init        ::= ScalarInit Literal
              | VectorInit Literal list


Statements

Abstract Syntax

Stmt        ::= Assign     { value : Expr,  target : LValue }
              | When       { cond  : Expr,  then   : Stmt list,
                             els   : Stmt list (* empty if no other| *) }
              | Repeat     { cond  : Expr,  body   : Stmt list }
              | Iterate    { var   : VarDecl, step : Step,
                             bound : BoundOp * Expr, body : Stmt list }
              | Back       { value : Expr }
              | Pipeline   PipelineStmt
              | VDecl      VarDecl
              | ExprStmt   Expr
              | Break

Step        ::= Incr | Decr                           (* ++ or -- *)
BoundOp     ::= Lt | Le | Gt | Ge            (* <  <=  >  >= *)

Pipeline Statement
Abstract Syntax

PipelineStmt ::= PipelineStmt {
                    source  : PipeSource,
                    steps   : PipeStep list,
                    sink    : PipeSink option
                  }

PipeSource   ::= StdIn                            (* 'in' keyword *)
               | ExprSrc Expr                         (* expression as source *)

PipeStep     ::= BindTo  LValue              (* > variable *)
               | CallFn  Ident                         (* > function name *)
               | CallMem Ident Ident           (* > obj.method *)

PipeSink     ::= StdOut                            (* 'out' keyword *)


Expressions & L-Values

Abstract Syntax

Expr        ::= Lit      Literal
              | LVal     LValue
              | BinOp    { op : BinOp, left : Expr, right : Expr }
              | UnOp     { op : UnOp,  arg  : Expr }
              | Deref    Expr                          (* *expr for generics *)
              | Group    Expr

LValue      ::= Var     Ident                         (* simple variable    *)
              | VecElem  { arr : Ident, idx : Expr }  (* a|i|               *)
              | Member   { obj : Ident, field : Ident }(* N.a               *)

BinOp       ::= Add | Sub | Mul | Div | Mod
              | Eq | Ne | Lt | Le | Gt | Ge

UnOp        ::= Neg | Not | PreInc | PreDec | PostInc | PostDec

Literal     ::= IntLit  Int
              | FltLit  Float
              | ChrLit  Char
              | StrLit  String
              | BoolLit Bool

Type        ::= Int | Float | Char | Str | Bool
              | Vec    { elem : Type, size : Expr }   (* T|n|  *)
              | Named  Ident                          				 (* class or type param *)
              | File


4. Type System

Semantica is statically and manifestly typed. All variable and parameter types are declared explicitly. There is no implicit coercion between primitive types.

4.1 Type Rules — Expressions

INT-LIT		⊢ n : int		FLOAT-LIT	⊢ f : float
CHAR-LIT	⊢ c : char	BOOL-LIT	⊢ b : bool

VAR			
Γ(x) = T	 
──────
Γ ⊢ x : T

VEC-ACCESS		
Γ ⊢ a : Vec(T,n)	Γ ⊢ i : int	   
────────────────
Γ ⊢ a|i| : T

ARITH			
Γ ⊢ e₁ : T	Γ ⊢ e₂ : T	T ∈ {int,float}  
────────────────────────
Γ ⊢ e₁ ⊕ e₂ : T

EQ				
Γ ⊢ e₁ : T	Γ ⊢ e₂ : T 
──────────────  
Γ ⊢ e₁ == e₂ : bool

CMP (cond)		
Γ ⊢ e₁ : T	Γ ⊢ e₂ : T	T ordered   
──────────────────────
Γ ⊢ e₁ > e₂ : bool


4.2 Type Rules — Statements

ASSIGN			
Γ ⊢ e : T		Γ(x) = T   
─────────────
Γ ⊢ e > x ok

WHEN			
Γ ⊢ c : bool	Γ ⊢ s₁…sₙ ok
────────────────
Γ ⊢ when| c … |when ok

REPEAT			
Γ ⊢ c : bool	Γ ⊢ body ok
─────────────────
Γ ⊢ repeat| c … |repeat ok

BACK
Γ ⊢ e : T		T = returnType(currentSub)
────────────────────────
Γ ⊢ back| e |back ok


4.3 Type Rules — Declarations

SUB
Γ, params ⊢ body ok		body returns T
────────────────────────
Γ ⊢ sub| params > f : T ok

CLASS
Γ ⊢ fields ok		Γ ⊢ methods ok
────────────────────────
Γ ⊢ class| C … |class ok


4.4 Context-Sensitive Operator

The > operator — disambiguation rule

Γcond — typing context inside when|, repeat|, or iterate| guard

Γstmt — typing context inside statement position


If e₁ > e₂ appears in Γcond → comparison, type: bool

If e > lv appears in Γstmt → assignment, type: unit


4.5 Generics (Parametric Polymorphism)

Construct			Rule / Type							Notes
template| typename T |template			Introduces type variable T into scope of following sub	

sub| (T x) > f : T		Polymorphic function over T	

T (args) > f > result	Monomorphisation at call site — T bound to actual type
	
*first	Dereference of iterator; type is T when Iter = T*		C++-style pointer semantics


5. Operational Semantics

Big-step (natural) semantics. The judgment ⟨s, σ⟩ ⇓ σ′ reads: statement s in store σ reduces to updated store σ′.

5.1 Assignment

⟨e, σ⟩ ⇓ v lv resolves to location ℓ in σ
──────────────────────────────────────
⟨ e > lv, σ ⟩ ⇓ σ[ℓ ↦ v]

5.2 Conditional (when / other)
WHEN-TRUE

⟨cond, σ⟩ ⇓ true
⟨then, σ⟩ ⇓ σ′
────────────────────
⟨when| cond then |when, σ⟩ ⇓ σ′

WHEN-FALSE
⟨cond, σ⟩ ⇓ false
⟨els, σ⟩ ⇓ σ′
────────────────────
⟨when| cond then other| els |other |when, σ⟩ ⇓ σ′

5.3 Loops
REPEAT-STEP
⟨cond, σ⟩ ⇓ true
⟨body, σ⟩ ⇓ σ′
⟨repeat| cond body |repeat, σ′⟩ ⇓ σ″
──────────────────────────
⟨repeat| cond body |repeat, σ⟩ ⇓ σ″

REPEAT-DONE
⟨cond, σ⟩ ⇓ false
──────────────────────────
⟨repeat| cond body |repeat, σ⟩ ⇓ σ

5.4 Function Call & Return
CALL
args ⇓ v₁…vₙ
σ′ = σ ∪ [p₁↦v₁,…,pₙ↦vₙ] (fresh frame)
⟨body, σ′⟩ ⇓ σ″ with back|e|back encountered, e ⇓ vᵣ
───────────────────────────────
⟨args > f, σ⟩ ⇓ (vᵣ, σ) (* frame discarded *)


6. Pipeline Execution Model

The pipeline is the core distinguishing feature of Semantica. It desugars into a sequence of assignments and calls evaluated strictly left-to-right.

Construct			Rule / Type							Notes

in > x				Read one value from stdin, assign to x	
in > x, y				Read two values, assign left-to-right	
in > x > f				Read x, immediately pass to function f	
in > x > f > out		Read x → call f(x) → print result		canonical I/O idiom

e > x				Evaluate e, assign to x	assignment form
e > f					Evaluate e, call f(e), discard result	
e > f > out			Call f(e), print return value	
in > a, b > f > out		Multi-arg input, call, output in one expression	Semantica pipeline motif

Desugar Rules

EBNF

(* Pipeline desugaring — P[src, steps, sink] *)

P[ in,      [],                 		none  ] = ()
P[ in,      [BindTo x | rest],  sink  ] = x = read(); P[ x, rest, sink ]
P[ e,       [CallFn f | rest],  	sink  ] = let v = f(e); P[ v, rest, sink ]
P[ e,       [BindTo x | rest],  sink  ] = x = e; P[ x, rest, sink ]
P[ v,       [],                 		StdOut] = print(v)
P[ v,       [],                		 none  ] = v

(* Multi-bind: in > a, b > f > out *)
= a = read(); b = read(); result = f(a, b); print(result)

7. Scope & Name Resolution

Semantica uses static (lexical) scoping with four nested scope levels.

Construct			Rule / Type													Notes

Module scope		Names declared at the top level of module| … |module			visible to all subs in module

Intermodular scope	Names exported via sub_sig in intermodular| … |intermodular	visible across linked modules

Sub (function) scope	Parameters and locals of sub| … |sub							stack-allocated, discarded on return

Block scope	Variables declared inside when|, repeat|, iterate|						shadow outer names, cleaned on block exit

Class scope	Fields and methods inside class| … |class								accessed via obj.member dot notation

Template scope	typename params introduced by template| … |template				bound at monomorphisation site


Name Resolution Order

Block	Sub		Class (if in method)	Module	Intermodular

Shadowing
A name declared in an inner scope shadows any same-name binding in an outer scope for the duration of that inner scope. After the inner scope exits, the outer binding is restored. Shadowing produces no error; the outer binding becomes unreachable for the duration of the inner scope.

Module Linking

EBNF

(* Intermodular resolution *)

link| other.sem |link
  → loads module 'other', makes its exported SubSig names available

sub| T a, T b > sharedFn : T |sub  (* in intermodular| block *)
  → declares sharedFn is callable from, and defined in, this module's sub body
